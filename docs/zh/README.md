## 核心论点

系统管理员真正的功夫，从来不是把每个平台的每个服务都背下来。它是一套**可迁移的心智模型**，加上**在任何平台上快速上手的纪律**。到了 2026 年，后半句被 AI 极大加速了 —— 学习曲线从几个月压到几天，**前提是**你已经有了驾驭它、并在它出错时抓住它的判断力。

这个仓库就是拿三大云把这个想法验证一遍。对每个平台，都按同样的顺序回答三个问题：

1. **它是什么？** —— 平台的职责 + 核心积木，一页看懂。
2. **当它的管理员，到底要会做什么？** —— 一份具体、可勾选的胜任清单，不是营销功能列表。
3. **在 AI 辅助下怎么快速胜任？** —— 方法论：怎么**又快又能验证**，用 AI 而不是盲信 AI。

然后用**动手实验（labs）**逼你证明 —— 可以真跑、能拆掉的脚本和 IaC。

## 为什么说"都是同一套动作"

一旦你正经管过一个平台，下一个大多只是**在同一副骨架上换语法**：

> **注册一个带权限范围的身份 → 拿到 token / 凭证 → 用它的 API 和 IaC 驱动平台。**

Jamf、Intune、Entra、Configuration Manager 是这样；AWS、Azure、GCP 也是这样。这个仓库就吃这个红利：**把模式学透一次**（见 [`00-the-operating-model.md`](../../00-the-operating-model.md)），之后每个新平台都变成一道"用 AI 就能秒答的映射题"。
